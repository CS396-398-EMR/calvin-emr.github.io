Deployment

Deploying to Firebase Hosting

The Calvin EMR web application is hosted live on Firebase Hosting at https://calvin-emr.web.app. The app is configured with Firebase's GitHub Action feature for continuous deployment. The live site is built and deployed off of the main branch in our GitHub repository. Firebase also generates a live preview site for each pull request that is made. It is recommended to look at the pull request's preview before merging into the main branch, since the live preview shows what the actual site will look like after merging.

After a branch is merged into main, Firebase automatically builds and deploys the updated application.

You can read more about Firebase Hosting here.

Deploying Cloud Functions

The application uses Firebase Cloud Functions to handle user account creation and deletion. You can read more about Firebase Cloud Functions here.

To deploy the Cloud Functions, first install all node modules in the functions directory. Assuming you are in the root directory:

cd functions
nvm use v16.15.0
npm install

Then run the following command:

firebase deploy --only functions

This will run a linter on the files containing the function code to ensure the code is formatted correctly. This check may fail, so be sure to fix all errors that are displayed in the console before running the command again (thankfully, the errors are pretty informational).

After the code passes the formatting test, it will be packaged and deployed to Firebase. You can view the functions and the function logs under the Functions tab in the Firebase project.
