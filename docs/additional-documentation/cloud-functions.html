Cloud Functions

The application uses Firebase Cloud Functions to handle user account creation and deletion. The Cloud Functions run automatically in the backend based on certain triggers. We write their code in the functions directory and then compile and deploy them to Firebase. See the Workflow page for more information on deploying Cloud Functions.

The source code for these functions is contained in the functions/src/index.ts file.

Read more about Firebase Cloud Functions here.

createUser

Parameters:

  • data: object passed to the function. This object must contain an user field to store the User object that will be used to specify the new account's information.
  • context: object containing authentication information.

Function that creates an account with the data contained in data.user. The account configuration is specified as follows:

  • uid: the account's ID
  • email: the account's email
  • password: composed of the account's email prefix and the account's ID
  • displayName: the account's first name and last name
  • disabled: false

The function then creates the corresponding User document in Firestore with the account ID as the document ID. The information in the document is taken from data.user.

This function will not run if the user calling it is not authenticated.

deleteUser

Parameters:

  • data: object passed to the function. This object needs to contain an id field to store the ID of the user/account to delete.
  • context: object containing authentication information.

Function that deletes the account specified by data.id. This function will not run if the user calling it is not authenticated.

userDeleted

Trigger that runs each time a user's authentication account is deleted. The function takes the ID from the deleted account and uses it to delete the corresponding User document in Firestore.
